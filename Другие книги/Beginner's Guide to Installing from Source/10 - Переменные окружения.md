**Параметры для настройки компиляции и установки приложения обычно передаются в качестве параметров командной строки скрипту «`configure`»** или программе `make`. Однако иногда параметры конфигурации передаются **через переменные окружения.** Их можно задать, поместив определения в начало команды, например:

```bash
NAME=tom ENABLE_FOO=no ./configure
```

Переменные окружения также могут быть определены перед запуском команды:

```bash
export NAME=tom
export ENABLE_FOO=no
./configure
```

Какие параметры доступны, обычно описывается в файлах README или INSTALL архива или на веб-сайте проекта. Иногда доступные параметры можно увидеть, запустив `./configure --help`.

### Зачем вам это нужно?

**Гибкость настройки:**

- Переменные окружения — ещё один способ повлиять на сборку.

- **Для LFS:** Вы будете использовать множество переменных окружения (например, `CFLAGS`, `LDFLAGS`) для настройки компилятора.
